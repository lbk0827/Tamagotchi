# v0.1 개발 체크리스트 (AI Pet)

기준 문서: `Docs/MVP_AiPet.md`

## 0) v0.1 범위 잠금
- [ ] 타겟 OS 1개 확정 (macOS 또는 Windows)
- [ ] 펫 아트 스타일 확정 (픽셀/벡터)
- [ ] AI 말투 1개 확정 (친구형/동생형/코치형)
- [ ] Out of Scope 항목 재확인 (멀티펫/음성/상점 제외)

## 1) 프로젝트 세팅
- [x] Electron + TypeScript 프로젝트 초기화
- [x] 개발/빌드 스크립트 구성 (`dev`, `build`)
- [x] 기본 폴더 구조 생성 (`main`, `renderer`, `assets`, `data`)
- [x] 환경변수 로딩 구조 추가 (`.env`, API Key)

## 2) 알 모양 플로팅 윈도우
- [x] Borderless + Transparent + Always-on-top 설정
- [x] 알 모양 UI 렌더(배경 스킨)
- [ ] 클릭 가능한 영역(히트 테스트) 정리
- [x] 드래그 이동 동작 구현
- [x] 닫기/최소화 대체 UX 구현(아이콘 모드 또는 메뉴)

## 3) LED 패널 + 기본 UI
- [x] 알 내부 사각 LED 패널 구현
- [x] 펫 렌더링 영역 고정
- [x] 액션 버튼 4개 구현 (Feed, Play, Clean, Sleep)
- [x] 상태 표시 UI 구현 (Hunger, Mood, Energy, Cleanliness, Health)
- [x] 말풍선 텍스트 영역 구현

## 4) 상태 시스템
- [x] 펫 상태 모델 정의 (5개 스탯, 0~100)
- [x] 시간 경과 Tick 시스템 구현
- [x] 액션별 수치 변화 규칙 구현
- [x] 상태 판정 규칙 구현 (Happy, Normal, Tired, Sick)
- [x] 방치 패널티(Health 감소) 연결

## 5) 애니메이션/반응
- [x] Idle 애니메이션
- [x] Happy 애니메이션
- [x] Sleep 애니메이션
- [x] Sick 애니메이션
- [x] 상태 전환 시 애니메이션/말풍선 반응 연결

## 6) AI 대화
- [x] 텍스트 입력창 + 전송 UX 구현
- [x] LLM API 호출 모듈 구현
- [x] 프롬프트 템플릿 구성 (펫 성격 + 현재 상태 반영)
- [x] 응답 길이 제한(짧은 답변)
- [x] API 실패 시 폴백 대사 처리

## 7) 저장/복구
- [x] 로컬 JSON 저장 모듈 구현
- [x] 저장 항목 연결 (상태값, 마지막 접속 시각, 펫 이름, 성격)
- [x] 앱 종료/주기 저장 트리거 구현
- [x] 앱 시작 시 복구 로직 구현
- [x] 오프라인 경과 시간 반영 로직 구현

## 8) 안정화

- [ ] 로그 레벨 정리(개발/배포)
- [ ] 성능 점검(Idle CPU/메모리)
- [ ] UI 끊김/클릭 버그 수정

## 9) QA 체크 (수용 기준 기반)

- [ ] 4개 액션이 상태값에 즉시 반영

- [ ] AI 응답 실패 시 폴백 문구 정상 노출
- [ ] 재실행 후 저장 데이터 복구
- [ ] 30분 방치 테스트 통과
- [ ] 앱 10회 재시작 시 치명 오류 없음
