# MVP: Desktop Floating AI Tamagotchi

## 1. 제품 개요
- 제품명(가칭): AI Pet
- 목표: 데스크톱 위에 항상 떠 있는 알 모양의 다마고치 UI 안에서 AI 반려 펫과 상호작용한다.
- 핵심 컨셉:
  - 일반 사각 윈도우가 아닌 알(egg) 형태의 플로팅 창
  - 알 내부에 레트로 감성의 사각 LED 화면
  - LED 화면 안에서 펫이 상태 변화, 애니메이션, 대화를 수행

## 2. MVP 목표
- 외형적으로 "다마고치 같다"는 인상을 준다.
- 기본 육성 루프(먹이, 놀이, 수면, 청결) 1사이클이 동작한다.
- 펫과 짧은 자연어 대화가 가능하다.
- 앱 재시작 후에도 상태가 유지된다.

## 3. MVP 범위 (In Scope)

### 3.1 윈도우/셸
- Always-on-top 플로팅 창
- 무테(borderless) + 투명 배경 지원
- 알 모양 마스크(원형/타원형 기반)
- 드래그로 이동
- 닫기/최소화 대체(작은 아이콘 모드 또는 우클릭 메뉴)

### 3.2 UI 구성
- 알 본체(스킨)
- 내부 사각 LED 패널
- LED 내부 펫 렌더 영역
- 기본 액션 버튼 4개:
  - Feed(먹이)
  - Play(놀이)
  - Clean(청결)
  - Sleep(수면)
- 상태 표시(아이콘 또는 작은 바): Hunger, Mood, Energy, Cleanliness, Health

### 3.3 펫 상태 시스템
- 상태값 5종: Hunger, Mood, Energy, Cleanliness, Health (0~100)
- 시간 경과에 따른 자동 감소/증가 규칙
- 임계치 기반 상태 판정:
  - Happy / Normal / Tired / Sick
- 액션별 상태 변화 규칙
- 과도한 방치 시 Health 감소

### 3.4 펫 행동/애니메이션
- 최소 상태 애니메이션 4종:
  - Idle
  - Happy
  - Sleep
  - Sick
- 말풍선/짧은 텍스트 반응

### 3.5 AI 대화
- 입력: 짧은 텍스트(1~2문장)
- 출력: 펫 캐릭터 톤의 짧은 응답
- 상태 연동:
  - 배고프면 "배고프다" 성격 반영
  - 졸리면 응답 길이/톤 변화
- 안전장치:
  - 응답 실패 시 로컬 폴백 대사 사용

### 3.6 저장/복구
- 로컬 저장(JSON)
- 저장 항목:
  - 상태값
  - 마지막 접속 시각
  - 펫 이름
  - 성격 파라미터(초기 버전)
- 재실행 시 오프라인 경과 시간 반영

## 4. MVP 제외 범위 (Out of Scope)
- 멀티 펫
- 온라인 동기화/계정 시스템
- 음성 인식/음성 합성
- 복잡한 진화 트리(다단계)
- 상점/재화/인앱결제
- 모바일 동시 출시

## 5. 기술 제안 (1차)
- 데스크톱 프레임워크: Electron
- UI: HTML/CSS/TypeScript
- 렌더링: CSS 애니메이션 또는 Canvas
- AI 연동: LLM API 1종 (추후 교체 가능 구조)
- 데이터 저장: 로컬 파일(JSON)

선정 이유:
- 무테/투명/항상 위 창 구현이 비교적 쉬움
- UI 반복 실험 속도가 빠름
- AI API 연결이 단순함

## 6. 사용자 시나리오
1. 사용자가 앱 실행
2. 바탕화면에 알 모양 창이 나타남
3. LED 화면 안에 펫이 Idle 상태로 등장
4. 사용자가 Feed 클릭 -> Hunger 감소, Mood 소폭 증가
5. 사용자가 "오늘 어땠어?" 입력
6. 펫이 현재 상태를 반영해 짧게 대답
7. 앱 종료 후 재실행해도 상태가 유지됨

## 7. 성공 기준 (Acceptance Criteria)
- 알 모양 플로팅 창이 macOS/Windows 중 최소 1개 OS에서 안정 동작
- 4개 기본 액션이 상태값에 즉시 반영
- 시간 경과 로직이 정상 적용
- AI 응답 성공률 95% 이상(실패 시 폴백 응답 표시)
- 재시작 후 저장 데이터 복원 성공

## 8. 리스크 및 대응
- 리스크: 플랫폼별 투명/클릭영역 처리 차이
  - 대응: MVP는 1개 OS 우선 고정 후 추후 확장
- 리스크: API 지연/실패
  - 대응: 타임아웃 + 폴백 대사
- 리스크: UI는 예쁘나 육성 재미 부족
  - 대응: 상태 변화 피드백(표정/말풍선) 강화

## 9. 2주 개발 플랜 (예시)
- Day 1-2: 프로젝트 셋업, 알 모양 창 프로토타입
- Day 3-4: LED 패널 + 펫 기본 렌더
- Day 5-6: 상태 시스템 + 4개 액션 연결
- Day 7-8: 저장/복구 + 오프라인 경과시간 반영
- Day 9-10: AI 대화 연결 + 폴백
- Day 11-12: 안정화/버그 수정
- Day 13-14: 데모 빌드 + 문서 정리

- 확정 후 바로 착수:
